The Problem

Client-side proving is slow.
Server-side proving leaks your data.

Aztec transactions require zero-knowledge proofs. Generating them in the browser takes minutes. Outsourcing to a server sacrifices privacy. TEE-Rex solves both.

How It Works

Three modes. One API.

Local (WASM)

Prove entirely in the browser using bb.js WebAssembly. Fully private, no server needed.

Slow Fully Private

UEE (Server)

Offload proving to a remote server running native bb. Fast proofs, but the server sees your data.

Fast Not Private
Recommended

TEE (Nitro Enclave)

Prove inside an AWS Nitro Enclave. Hardware attestation guarantees your data stays private and the code is untampered.

Fast Private Attested
Architecture

What happens inside the enclave

1

Encrypt & Send

Your transaction witness is PGP-encrypted with the enclave's public key. Only the enclave can decrypt it.

2

Attest & Decrypt

The enclave generates a cryptographic attestation document proving its identity, then decrypts your data in isolated memory.

3

Prove & Return

Native bb generates the zero-knowledge proof inside the enclave. The proof is returned; your witness is discarded.

Client encrypt(witness, enclave_pk)
Nitro Enclave
→ Verify attestation
→ Decrypt witness
→ Generate ZK proof (native bb)
→ Discard witness
Client receives proof only
TypeScript SDK

Drop-in replacement

Replace your Aztec prover with TEE-Rex in one line. The SDK handles encryption, attestation verification, and proof retrieval.

app.ts 
import { TeeRexProver } from "@alejoamiras/tee-rex"

// One constructor — mode is auto-detected from the server
const prover = new TeeRexProver("https://tee-rex.dev")

// Use it anywhere you'd use an Aztec prover
const wallet = await getSchnorrAccount(pxe, secret, salt, {
  prover  // ← drop-in replacement
}).deploy().wait()
